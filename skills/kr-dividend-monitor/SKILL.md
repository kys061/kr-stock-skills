# kr-dividend-monitor: 배당 안전성 모니터링

> DART 공시 기반 5대 강제 리뷰 트리거(T1-T5)로 배당 안전성을 모니터링합니다.
> US kanchi-dividend-review-monitor의 한국 적용 버전.

## 사용 시점

- 보유 배당주의 감배/무배 리스크를 조기에 감지할 때
- DART 공시를 기반으로 실적 악화를 모니터링할 때
- 배당 안전성 점수로 포트폴리오 건전성을 평가할 때

## 방법론

### 5대 강제 리뷰 트리거 (T1-T5)

| 트리거 | 설명 | 심각도 | 소스 |
|--------|------|:------:|------|
| T1 | 감배 감지 (DPS 감소) | CRITICAL | DART |
| T2 | 무배당 전환 | CRITICAL | DART |
| T3 | 실적 악화 (영업이익 -50% or 적자) | HIGH | DART |
| T4 | 배당성향 위험 (100% 초과) | HIGH | 계산 |
| T5 | 지배구조 이슈 | MEDIUM | DART |

### 상태 머신

```
OK → WARN → REVIEW → EXIT_REVIEW
```

### 배당 안전성 스코어 (0-100)

| 컴포넌트 | 가중치 | 만점 기준 |
|---------|:------:|----------|
| 배당성향 | 30% | ≤50% |
| 실적 안정성 | 25% | 5년 연속 흑자 |
| 배당 이력 | 25% | 10년 연속 배당 |
| 부채 건전성 | 20% | 부채비율 ≤80% |

## 한국 시장 적응

- 공시 소스: SEC → **DART 전자공시**
- 트리거 감지: 8-K → **주요사항보고서/정기보고서**
- 감사의견: PCAOB → **금감원 감사보고서**
- 지배구조: Proxy → **5% 대량보유 공시**

## 실행 방법

```

```

## Output Rule (마크다운 리포트 저장)

- 분석 완료 후 결과를 **마크다운 파일로 저장**한다
- 저장 경로: `reports/{skill_name}_{identifier}_{name}_{YYYYMMDD}.md`
  - `{identifier}`: 종목코드 (없으면 `market`)
  - `{name}`: 종목명 또는 분석 대상명 (한글 가능)
  - 예: `reports/kr-stock-analysis_005930_삼성전자_20260305.md`
- 파일 구조:
  ```markdown
  # {스킬명} 분석 리포트
  > 생성일: {YYYY-MM-DD HH:MM} | 대상: {identifier}

  ## 요약
  (핵심 결론 2-3줄)

  ## 상세 분석
  (스킬별 분석 결과를 마크다운 테이블, 리스트, 헤더로 구조화)

  ## 판단 근거
  (점수, 등급, 시그널 등 정량 데이터)

  ---
  *Generated by {skill_name}*
  ```
- `reports/` 디렉토리가 없으면 자동 생성
- 동일 파일명이 존재하면 덮어쓰기 (같은 날 재분석 시)

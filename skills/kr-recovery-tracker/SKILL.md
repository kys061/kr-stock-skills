# kr-recovery-tracker: 폭락 후 회복 추적

> 시장 급락 이후 **7개 핵심 지표**를 추적하여 회복 진행 상황을 모니터링합니다.
> **5단계 회복 모델**로 현재 시장이 어디까지 왔는지 판단합니다.
> 매일 동일 형식으로 추적하여 **어제 대비 변화**를 비교할 수 있습니다.

## 사용 시점

- 시장 급락(KOSPI -5%+ 일간 하락) 발생 후 회복 추적
- 위기 이벤트(지정학/금융위기/팬데믹) 후 일일 모니터링
- 회복 단계 판단과 3단계(안정화) 진입 조건 체크
- 이전 리포트 대비 개선/악화 추적

## 데이터 소스 우선순위

```
1순위: yfinance (KOSPI, VIX, EWY, USD/KRW — 자동 수집)
2순위: WebSearch (CNN F&G, 외국인 수급, 뉴스 헤드라인)
3순위: WebFetch (특정 URL 직접 접근)
```

### 주요 데이터 소스

| 지표 | 1순위 소스 | 2순위 소스 |
|------|-----------|-----------|
| KOSPI | yfinance `^KS11` + RSI 계산 | Investing.com |
| VIX | yfinance `^VIX` | CNBC, CBOE |
| CNN F&G | WebSearch | CNN Markets |
| USD/KRW | yfinance `KRW=X` | Trading Economics |
| EWY | yfinance `EWY` | Yahoo Finance |
| 외국인 수급 | WebSearch (KRX 뉴스) | 네이버 증권 |
| 뉴스 톤 | WebSearch | CNBC, 네이버 |

## 추적 지표 (7개)

| # | 지표 | 카테고리 | 추적 항목 |
|:-:|------|---------|----------|
| 1 | KOSPI | 지수 | 종가, 바닥 대비 변동률, 위기전 대비 변동률, RSI(14) |
| 2 | VIX | 변동성 | 종가, 위기전 대비 변동률 |
| 3 | CNN F&G | 심리 | 지수(0-100), 구간명 |
| 4 | USD/KRW | 환율 | 종가, 위기전 대비 변동률 |
| 5 | EWY | 한국 프록시 | 종가, 위기전 대비 변동률 |
| 6 | 외국인 수급 | 수급 | 일간/주간 순매수/순매도 |
| 7 | 뉴스 톤 | 센티먼트 | 긍정/부정/중립 비율, 주요 헤드라인 |

## 5단계 회복 모델

| 단계 | 명칭 | 주요 특징 |
|:----:|------|----------|
| 1 | 패닉 | VIX 35+, 대폭 하락, 마진콜, 서킷브레이커 |
| 2 | 기술적 반등 | 숏커버 + 반발매수, V자 반등, VIX 25-35 |
| 3 | 안정화 | VIX < 25, 환율 안정, 외국인 순매수 전환 |
| 4 | 추세 회복 | 주요 MA 회복, 거래량 정상화, 업종 로테이션 |
| 5 | 완전 회복 | 위기전 수준 복귀, VIX < 20, 신고가 시도 |

### 3단계(안정화) 진입 조건

| 조건 | 기준 |
|------|------|
| VIX | < 25 |
| USD/KRW | < 위기전 +3% |
| CNN F&G | > 35 (Fear 이상) |
| 외국인 수급 | 순매수 전환 |
| KOSPI 회복률 | > 60% |
| 일간 변동성 | < 3% |

> 6개 중 최소 4개 충족 시 3단계 진입으로 판단

## 위기 컨텍스트 설정

리포트 작성 전 아래 정보를 사용자로부터 받거나 이전 리포트에서 가져온다:

| 항목 | 설명 |
|------|------|
| 위기 원인 | 이벤트명 (예: 이란-미국 군사충돌) |
| 위기 시작일 | 최초 급락일 |
| 위기전 기준일 | 마지막 정상 거래일 |
| 위기전 KOSPI | 기준일 종가 |
| 바닥 KOSPI | 최저점 종가 및 일자 |

## 스크립트 실행 (1순위)

yfinance로 자동 수집 가능한 4개 지표(KOSPI, VIX, EWY, USD/KRW)를 먼저 수집한다.

```bash
cd ~/stock/skills/kr-recovery-tracker/scripts
python3 recovery_tracker.py                          # 전체 수집
python3 recovery_tracker.py --pre-crisis 6244        # 위기전 KOSPI 지정
python3 recovery_tracker.py --bottom 5094            # 바닥 KOSPI 지정
python3 recovery_tracker.py --format json            # JSON 출력
```

나머지 3개 지표(CNN F&G, 외국인 수급, 뉴스 톤)는 WebSearch로 보완한다.

### 정적 참조 데이터

`references/recovery_stages.json`에 5단계 정의와 3단계 진입 조건을 보관한다.
스크립트가 자동 로드하여 단계 판정에 활용.

### 오류 핸들링

| 상황 | 대응 |
|------|------|
| yfinance 타임아웃 | 해당 지표 `error` 표시, 나머지 계속 수집 |
| 데이터 부족 (< 2일) | `Insufficient data` 반환, RSI 계산 생략 |
| references 파일 누락 | 기본 하드코딩 값 사용, 경고 출력 |
| 위기전/바닥 미지정 | 3개월 내 최고/최저로 자동 추정 |

## 출력 형식

### 1. 회복 대시보드 (필수)

```markdown
## 회복 대시보드

| 항목 | 수치 | 변화 | 상태 | 출처 |
|------|------|------|:----:|------|
| **KOSPI** | {값} | 바닥 대비 {%}, 위기전 대비 {%} | {신호등} | yfinance |
| **VIX** | {값} | 위기전 대비 {%} | {신호등} | yfinance |
| **CNN F&G** | {값} ({구간}) | 변화 설명 | {신호등} | CNN Markets |
| **USD/KRW** | {값} | 위기전 대비 변화 | {신호등} | yfinance |
| **EWY** | ${값} | 위기전 대비 {%} | {신호등} | yfinance |
| **외국인 수급** | {순매수/순매도 금액} | 요약 | {신호등} | KRX·뉴스 |
| **뉴스 톤** | {긍정N/부정N} | 요약 | {신호등} | CNBC 등 |
```

### 2. KOSPI 회복률 계산 (필수)

```markdown
### KOSPI 회복률 계산

위기전 (M/D 종가):  {값}
바닥   (M/D 종가):  {값}
현재   (M/D 종가):  {값}
──────────────────────────────
총 하락폭:         {값} ({%})
현재 회복폭:       {값} ({%})
회복 비율:         {%} (= 회복폭 / 총 하락폭)
잔여 손실:         {%} (현재 → 위기전 수준)
```

### 3. 어제 대비 변화 (필수)

```markdown
## 어제 대비 변화

### 개선된 것
- 항목 1
- 항목 2

### 악화된 것
- 항목 1
- 항목 2

### 변화 없음
- 항목 1
```

### 4. 뉴스 헤드라인 요약 (필수, 3개)

```markdown
## 뉴스 헤드라인 요약 (3개)

| # | 헤드라인 | 톤 | 시장 영향 |
|:-:|---------|:--:|----------|
| 1 | **제목** | 긍정/부정 | 영향 설명 |
```

### 5. 회복 단계 판단 (필수)

```markdown
## 회복 단계 판단

1단계. 패닉          ████████████████████ ← (해당 시)
2단계. 기술적 반등    ████████████████
3단계. 안정화         ░░░░░░░░░░░░
4단계. 추세 회복      ░░░░░░░░
5단계. 완전 회복      ░░░░

### 현재: **N단계 ({명칭})** — {한 줄 근거}

### 다음 단계 진입 조건
| 조건 | 현재 | 필요 수준 | 충족 |
```

## 실행 방법

```
/kr-recovery-tracker
/kr-recovery-tracker 회복 추적 업데이트
/kr-recovery-tracker 위기전 KOSPI 6244 바닥 5094
```

### 인수 파싱

| 인수 | 동작 |
|------|------|
| (없음) | 전체 7개 지표 수집 + 회복 판단 |
| 위기전/바닥 수치 | 기준점 지정 |
| `간단` / `brief` | 대시보드 + 회복 단계만 출력 |

## 관련 스킬

| 스킬 | 관계 |
|------|------|
| kr-rebound-signal | 14개 반등 시그널 YES/NO (패닉→반등 전환점 탐지) |
| kr-indicator-deep-dive | 개별 지표 5항목 심층 분석 |
| kr-crisis-compare | 역대 위기와 유사도 비교 |

### 스킬 간 차이점

| 스킬 | 초점 | 시점 | 출력 |
|------|------|------|------|
| daily-market-check | 6개 지표 수치 확인 | 평시 | 상태 판정 |
| kr-rebound-signal | 반등 시그널 탐지 | 급락 직후 | YES/NO 체크 |
| **kr-recovery-tracker** | **회복 진행 추적** | **급락 후 수일~수주** | **회복률+단계 판단** |
| kr-crisis-compare | 역대 위기 비교 | 위기 발생 시 | 유사도 % |

## Output Rule (마크다운 리포트 저장)

- **템플릿**: `_kr_common/templates/report_macro.md` 구조를 따른다
- **공통 규칙**: `_kr_common/templates/report_rules.md` 참조
- 저장 경로: `reports/recovery-tracker_{YYYYMMDD}.md`
- `reports/` 디렉토리가 없으면 자동 생성
- 동일 파일명이 존재하면 덮어쓰기 (같은 날 재분석 시)

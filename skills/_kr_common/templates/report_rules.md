# Report Formatting Rules (공통 포맷팅 규칙)

> 모든 스킬이 마크다운 리포트를 생성할 때 반드시 준수하는 규칙.
> 유형별 템플릿: report_stock.md / report_screener.md / report_macro.md

---

## 1. 파일명 규칙

```
reports/{skill_name}_{identifier}_{name}_{YYYYMMDD}.md
```

| 리포트 유형 | identifier | name | 예시 |
|------------|-----------|------|------|
| 개별 종목 | 종목코드 | 종목명 | `kr-stock-analysis_005930_삼성전자_20260306.md` |
| 스크리닝 | `market` | 대상명 | `kr-canslim-screener_market_KOSDAQ_Top5_20260306.md` |
| 매크로/전략 | `market` | 생략 가능 | `kr-market-environment_market_20260306.md` |

- `reports/` 디렉토리가 없으면 자동 생성
- 동일 파일명은 덮어쓰기 (같은 날 재분석 시)

## 2. 공통 헤더

모든 리포트는 아래 헤더로 시작한다:

```markdown
# {skill_name} 리포트: {대상명} ({identifier})

> 생성일: {YYYY-MM-DD HH:MM} | 대상: {대상명} ({identifier})
> 데이터 소스: {Tier N 소스명} | 분석 유형: {COMPREHENSIVE/SCREENING/MACRO}
```

### 데이터 소스 표기

| 상태 | 표기 |
|------|------|
| KRX API 사용 | `KRX Open API (Tier 0)` |
| yfinance 사용 | `yfinance (Tier 1)` |
| 복합 소스 | `yfinance (Tier 1) + WebSearch (Tier 4)` |
| WebSearch 전용 | 아래 경고 블록 추가 |

WebSearch 폴백 시 반드시 추가:
```markdown
> **정밀도 한계**: 기술적 지표(RSI/MACD 등) 정확값 미확인, 뉴스/리포트 기반 추정치
```

## 3. 요약 섹션 (필수)

헤더 직후 `## 요약` 섹션을 배치한다.

- 핵심 결론 2-3문장
- 스코어링 스킬인 경우 요약 테이블 포함:

```markdown
## 요약

| 항목 | 값 |
|------|:---:|
| **종합 점수** | **{N} / 100** |
| **투자 등급** | **{GRADE}** |
| **현재가** | {price}원 |
```

## 4. 섹션 넘버링

- 최상위: `## 1.`, `## 2.`, ...
- 하위: `### 1.1`, `### 1.2`, ...
- 점수가 있는 섹션: `## 2. 펀더멘털 분석 -- {N}점` (헤딩에 점수 포함)

## 5. 테이블 정렬

| 데이터 유형 | 정렬 | 마크다운 |
|------------|:----:|---------|
| 텍스트 | 좌측 | `|------|` |
| 숫자/점수 | 우측 | `|-----:|` |
| 등급/상태 | 중앙 | `|:----:|` |

## 6. Sources (필수)

문서 최하단에 배치. `*Generated by*` 뒤에 위치:

```markdown
---
*Generated by {skill_name}*

Sources:
- [소스명1](URL1)
- [소스명2](URL2)
```

## 8. 이메일 발송 (선택)

리포트 MD 파일 생성 후 `.env`의 `EMAIL_ENABLED=true` 설정 시 자동 발송한다.

### 발송 절차

1. Write 도구로 `reports/` 에 MD 파일 생성
2. Bash로 이메일 발송 호출:
```bash
cd ~/stock && python3 skills/_kr_common/utils/email_sender.py "{리포트_파일_경로}" "{스킬명}"
```
3. 발송 실패 시에도 리포트 생성은 정상 완료로 간주 (fail-safe)

### 설정

`.env` 파일에 아래 항목을 설정한다 (Gmail App Password 방식):

| 항목 | 설명 |
|------|------|
| `EMAIL_ENABLED` | `true`/`false` (기본: false) |
| `EMAIL_SMTP_HOST` | `smtp.gmail.com` |
| `EMAIL_SMTP_PORT` | `587` |
| `EMAIL_SMTP_USER` | Gmail 주소 |
| `EMAIL_SMTP_PASSWORD` | 앱 비밀번호 (16자리) |
| `EMAIL_TO` | 수신자 주소 |

### 이메일 제목 자동 생성

파일명에서 자동 파싱:
```
[KR-Stock] {skill_name}: {대상명} ({YYYY-MM-DD})
```

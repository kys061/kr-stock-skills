# kr-sector-analyst

> 한국 업종 분석 스킬 - 업종별 성과 차트를 분석하여 경기 순환 위치와 로테이션 시나리오를 도출

## 개요

사용자가 제공한 **업종별 성과 차트 이미지**를 분석하여 현재 한국 시장의 경기 순환 국면을 판정하고,
향후 업종 로테이션 시나리오를 확률적으로 제시한다.

- **데이터 소스**: 사용자 제공 차트 이미지 (1주일/1개월 타임프레임)
- **API 불필요**: 순수 차트 분석
- **출력**: Markdown 분석 리포트

## 실행 조건

이 스킬은 사용자가 **한국 업종별 성과 차트 이미지**를 제공할 때 사용한다.
예: KRX 업종 지수 성과 비교, 네이버 금융 업종별 등락률, 증권사 업종 비교 차트

## 분석 워크플로우

### Step 1: 차트 이미지 수신 및 확인
- 차트 수, 타임프레임 (1주/1개월) 확인
- KRX 업종 분류 체계와 매핑

### Step 2: 레퍼런스 로드
```
Read references/kr_sector_rotation.md
```

### Step 3: 체계적 분석 (6단계)

#### 3.1 성과 패턴 관찰
- 1주일 vs 1개월 성과 비교
- 업종별 상대 성과 순위
- 같은 그룹(경기민감/방어) 내 일관성 확인

#### 3.2 경기 순환 국면 판정

| 국면 | 선도 업종 | 후행 업종 | 한국 특징 |
|------|-----------|-----------|-----------|
| 초기 회복 | 반도체, 자동차, 건설 | 통신, 유틸리티 | 외국인 순매수 재개 |
| 중기 확장 | IT, 2차전지, 금융 | - (광범위 상승) | 원화 강세 |
| 후기 과열 | 조선, 원자재, 은행 | 성장주 | 원자재 가격 상승 |
| 침체 | 통신, 유틸리티, 필수소비재 | 경기민감주 | 외국인 순매도 |

#### 3.3 한국 업종 그룹 분류

**경기민감 (Cyclical)**:
반도체, 자동차, 철강, 화학, 건설, 조선, 기계

**방어 (Defensive)**:
통신, 유틸리티, 필수소비재, 제약/바이오

**성장 (Growth)**:
2차전지, 인터넷/플랫폼, IT서비스, 엔터/콘텐츠

**금융 (Financial)**:
은행, 보험, 증권

**원자재/에너지 (Commodity)**:
정유, 해운, 비철금속

#### 3.4 시나리오 개발

**확률 범위 프레임워크**:
- 70-85%: 강한 근거, 복수 확인 신호
- 50-70%: 중간 근거, 일부 확인
- 30-50%: 약한 근거, 상충 신호
- 15-30%: 추측, 반대 지표

각 시나리오 포함:
- 시나리오명
- 확률 (합계 ~100%)
- 설명
- 수혜 업종 / 피해 업종
- 확인 촉매

#### 3.5 한국 시장 특수 고려사항
- **반도체 슈퍼사이클**: 삼성전자+SK하이닉스 = KOSPI 시총 30%+
- **외국인 지분율**: KOSPI 외국인 30%+ → 수급이 방향 결정
- **원달러 환율**: 원 강세=내수↑, 원 약세=수출↑
- **밸류업 프로그램**: 금융/지주 재평가 흐름

## 섹터 성장 전망 (Growth Outlook)

14개 섹터별 TAM/CAGR 기반 성장 등급:

| 등급 | 섹터 | CAGR |
|:----:|------|:----:|
| S | 반도체 | 12% |
| A | 배터리, 전력기기, 바이오, 방산, 조선 | 7-15% |
| B | 자동차, 엔터 | 4-8% |
| C | 화학, 철강, 금융 | 2-3% |
| D | 건설, 통신, 유틸리티 | 1-2% |

`scripts/sector_growth_scorer.py`로 프로그래밍 접근 가능.

## 분석 원칙

1. **객관성 우선**: 데이터가 결론을 인도하게 한다
2. **확률적 사고**: 불확실성을 확률 범위로 표현
3. **복수 타임프레임**: 1주/1개월 비교로 확인
4. **상대 성과 집중**: 절대 수익률이 아닌 상대 성과
5. **한국 특수성**: 수출 주도, 반도체 비중, 외국인 수급 반영

## Output Rule (마크다운 리포트 저장)

- 분석 완료 후 결과를 **마크다운 파일로 저장**한다
- 저장 경로: `reports/{skill_name}_{identifier}_{YYYYMMDD}.md`
  - `{identifier}`: 종목코드, 섹터명, 또는 분석 대상 (없으면 `market`)
  - 예: `reports/kr-stock-analysis_005930_20260305.md`
- 파일 구조:
  ```markdown
  # {스킬명} 분석 리포트
  > 생성일: {YYYY-MM-DD HH:MM} | 대상: {identifier}

  ## 요약
  (핵심 결론 2-3줄)

  ## 상세 분석
  (스킬별 분석 결과를 마크다운 테이블, 리스트, 헤더로 구조화)

  ## 판단 근거
  (점수, 등급, 시그널 등 정량 데이터)

  ---
  *Generated by {skill_name}*
  ```
- `reports/` 디렉토리가 없으면 자동 생성
- 동일 파일명이 존재하면 덮어쓰기 (같은 날 재분석 시)

# kr-technical-analyst

> 한국 주식 기술적 분석 스킬 - 주간/일간 차트 이미지를 분석하여 확률적 시나리오 도출
> **KRClient 불가 시 WebSearch 폴백으로 분석을 계속한다.**

## 데이터 소스 우선순위

1순위: KRX Open API / KIS API → 2순위: KRClient (pykrx/FDR) → 3순위: WebSearch 폴백.
KRClient 실패 시 WebSearch로 데이터를 수집하여 분석을 계속하며, 리포트에 데이터 소스를 명시한다.

## 개요

사용자가 제공한 **주간(Weekly) 또는 일간(Daily) 차트 이미지**를 체계적으로 분석하여
추세, 지지/저항, 이동평균, 거래량 패턴을 파악하고 확률적 시나리오를 제시한다.

- **데이터 소스**: 사용자 제공 차트 이미지
- **API 불필요**: 순수 차트 분석 (선택적으로 KRClient 수급 보조 데이터)
- **출력**: Markdown 분석 리포트

## 실행 조건

이 스킬은 사용자가 **한국 주식 차트 이미지**를 제공하고 기술적 분석을 요청할 때 사용한다.

## 분석 워크플로우

### Step 1: 차트 이미지 수신
- 종목명, 타임프레임 확인
- 차트 수 확인 (복수 차트 가능)

### Step 2: 레퍼런스 로드
```
Read references/technical_analysis_framework.md
```

### Step 3: 체계적 분석 (6단계)

#### 3.1 추세 분석 (Trend Analysis)
- **추세 방향**: 상승/하락/횡보
- **추세 강도**: 강/중/약
- **패턴**: 고점-저점 시퀀스 (Higher High/Higher Low 또는 Lower High/Lower Low)
- **탈진 신호**: 추세 전환 징후

#### 3.2 지지/저항 분석 (Support & Resistance)
- 수평 지지/저항 레벨
- 추세선 지지/저항
- 역할 전환 (저항→지지 또는 지지→저항)
- 컨플루언스 존 (복수 S/R 레벨 수렴)

#### 3.3 이동평균 분석 (Moving Average)
- 가격 vs 20일/50일/200일 MA 위치
- MA 정렬: 정배열(Bullish) / 역배열(Bearish) / 중립
- MA 기울기: 상승/하락/수평
- MA 크로스오버 (골든크로스/데드크로스)
- MA의 동적 지지/저항 역할

#### 3.4 거래량 분석 (Volume Analysis)
- 전체 거래량 추세
- 거래량 스파이크와 맥락
- 가격-거래량 확인/괴리
- 클라이맥스/탈진 패턴

#### 3.5 차트 패턴 & 가격 행동 (Patterns)
- **반전 패턴**: 해머, 유성형, 장악형, 피봇 등
- **지속 패턴**: 깃발, 삼각형, 쐐기 등
- **중요 캔들 포메이션**
- **돌파/붕괴** 이벤트

#### 3.6 시나리오 확률 배정

2-4개 시나리오 개발:
- **Base Case** (40-60%): 가장 가능성 높은 시나리오
- **Bull Case** (20-40%): 낙관적 시나리오
- **Bear Case** (20-40%): 비관적 시나리오
- **Alternative** (5-15%): 낮은 확률이나 가능한 시나리오

각 시나리오 포함:
- 시나리오명
- 확률
- 설명
- 지지 팩터 (2-3개 이상)
- 목표가
- 무효화 레벨

### Step 4 (선택): 한국 시장 보조 데이터

순수 차트 분석이 주(Primary)이며, 아래는 시나리오 확률 **보정용**(Secondary)으로만 사용:

```python
# 선택적 보조 데이터 (스크립트 실행 불필요 - SKILL.md 참조용)
# KRClient.get_investor_trading(ticker, start) → 외국인/기관 순매수 추세
# KRClient.get_short_selling(ticker, start) → 공매도 비율 변화
```

- **외국인 순매수 지속** → Bull 시나리오 확률 +5~10%
- **기관 순매도 전환** → Bear 시나리오 확률 +5~10%
- **공매도 잔고 급증** → Bear 시나리오 확률 +5%

## 한국 시장 가격제한폭 주의사항

### 상한가/하한가 캔들 해석
- **상한가 (+30%)**: 극도의 매수세. 다음 날 갭업 가능성. VI 발동 고려
- **하한가 (-30%)**: 극도의 매도세. 반등 시 데드캣 바운스 가능성

### 가격 변동폭 감안
- 한국 대형주 일일 변동폭: 보통 ±2~5%
- ±30%는 이론적 상한, 실제 소형주에서만 빈발
- US 대비 일일 변동폭 제한 → 캔들 길이 해석 보정

### VI (Volatility Interruption)
- 정적 VI: 전일 종가 대비 ±10% (KOSPI) / ±15% (KOSDAQ)
- 동적 VI: 직전 체결가 대비 ±3% (KOSPI) / ±6% (KOSDAQ)
- VI 발동 = 급변동 신호 → 차트에서 갭/긴 캔들

## 분석 원칙

1. **순수 차트 분석**: 뉴스, 펀더멘털, 감성 배제
2. **체계적 접근**: 모든 단계를 순서대로 수행
3. **객관적 평가**: 주관 배제, 관측 가능한 패턴만
4. **확률적 시나리오**: 절대적 예측 금지
5. **구체적 가격**: 모호한 표현 대신 정확한 가격 레벨 제시
6. **무효화 조건**: 각 시나리오의 실패 조건 명시

## Output Rule (마크다운 리포트 저장)

- 분석 완료 후 결과를 **마크다운 파일로 저장**한다
- 저장 경로: `reports/{skill_name}_{identifier}_{name}_{YYYYMMDD}.md`
  - `{identifier}`: 종목코드 (없으면 `market`)
  - `{name}`: 종목명 또는 분석 대상명 (한글 가능)
  - 예: `reports/kr-stock-analysis_005930_삼성전자_20260305.md`
- 파일 구조:
  ```markdown
  # {스킬명} 분석 리포트
  > 생성일: {YYYY-MM-DD HH:MM} | 대상: {identifier}

  ## 요약
  (핵심 결론 2-3줄)

  ## 상세 분석
  (스킬별 분석 결과를 마크다운 테이블, 리스트, 헤더로 구조화)

  ## 판단 근거
  (점수, 등급, 시그널 등 정량 데이터)

  ---
  *Generated by {skill_name}*
  ```
- `reports/` 디렉토리가 없으면 자동 생성
- 동일 파일명이 존재하면 덮어쓰기 (같은 날 재분석 시)

## 1. 배경 및 목적

사용자가 수동으로 관리하던 **주식종목선별 5가지 조건**을 자동화하여,
KOSPI/KOSDAQ 전 종목에서 조건 충족 종목을 자동 선별하는 스킬을 만든다.

### 원본 조건 (이미지 기반)

| # | 조건명 | 상세 |
|---|--------|------|
| 1 | 주가이평추세 | [일]0봉전 200일 이평 상승+보합추세 20회 유지 |
| 2 | 주가이평배열(4) | 200이평 < 150이평 < 50이평 < 종가 (정배열) |
| 3 | 52주 최저가대비 | 30% 이상 상승 |
| 4 | 52주 최고가대비 | -25% 이내 |
| 5 | 시가총액 | 현재가 기준 1,000억원 이상 |

> Minervini Stage 2 트렌드 템플릿과 유사하나, VCP 패턴/RS Rank 없이 **5조건 Pass/Fail** 방식.

### 기존 kr-vcp-screener 와 차이점

| 항목 | kr-vcp-screener | kr-stock-selector (신규) |
|------|----------------|------------------------|
| 목적 | VCP 브레이크아웃 대기 종목 | Stage 2 트렌드 유니버스 선별 |
| 조건 수 | 7 (Stage 2) + VCP 5컴포넌트 | 5 (순수 트렌드 템플릿) |
| RS Rank | 필수 (KOSPI 대비 >70) | 없음 |
| VCP 패턴 | 수축 횟수/깊이/Dry-Up | 없음 |
| 시가총액 | 필터 없음 | ≥ 1,000억원 필수 |
| 52주 최저 | +25% | +30% |
| 200MA 추세 | 22일 상승 | 20일 상승+보합 |
| 출력 | 스코어 0-100 | Pass/Fail + 통과 종목 리스트 |
| 복잡도 | High | Low (간결, 빠른 실행) |

## 3. 5가지 선별 조건 상세

### 조건 1: 주가이평추세 (200MA Trend)

```
200일 SMA가 최근 20거래일 동안 상승 또는 보합(flat) 유지
- 상승: SMA_today > SMA_yesterday
- 보합: |SMA_today - SMA_yesterday| < 0.001 * SMA_yesterday (0.1% 이내 변화)
- 20일 연속 "상승 OR 보합" → Pass
- 1일이라도 하락 → Fail
```

### 조건 2: 주가이평배열 — 4중 정배열

```
종가 > 50일 SMA > 150일 SMA > 200일 SMA
- 4개 값의 크기 순서 확인 (현재 거래일 기준)
- 하나라도 역전 → Fail
```

### 조건 3: 52주 최저가대비 +30% 이상

```
(현재 종가 - 52주 최저가) / 52주 최저가 ≥ 0.30
- 52주 = 최근 250거래일 (또는 365일 캘린더일)
- 52주 최저가: 기간 내 일봉 Low 최솟값
```

### 조건 4: 52주 최고가대비 -25% 이내

```
(현재 종가 - 52주 최고가) / 52주 최고가 ≥ -0.25
- 즉, 현재가 ≥ 52주 최고가 × 0.75
- 52주 최고가: 기간 내 일봉 High 최댓값
```

### 조건 5: 시가총액 ≥ 1,000억원

```
현재가 기준 시가총액 ≥ 100,000,000,000 원
- KRX Open API: get_market_cap() → 시가총액 칼럼
- 약 300~400개 종목이 대상 (사전 필터로 유니버스 축소)
```

## 4. 데이터 소스 아키텍처

### 2단계 수집 전략

```
Phase A: 유니버스 구축 (KRX Open API 1회 호출)
  → 전 종목 시가총액 조회
  → 시총 ≥ 1,000억원 필터
  → 대상 종목 ~300~400개 확정

Phase B: 개별 종목 분석 (yfinance batch download)
  → 대상 종목 300일 OHLCV 배치 다운로드
  → 50/150/200일 SMA 계산
  → 52주 고가/저가 계산
  → 5조건 판정
```

### 데이터 소스 우선순위

| 우선순위 | 소스 | 용도 | 비고 |
|:--------:|------|------|------|
| 1 | KRX Open API | 시가총액, 종목 리스트 | Tier 0 |
| 2 | yfinance | 일봉 OHLCV (300일) | Tier 1, 배치 가능 |
| 3 | pykrx | yfinance 실패 시 폴백 | Tier 2 |

### yfinance 배치 다운로드

```python
import yfinance as yf

# 한국 종목: 티커 + ".KS" (KOSPI) 또는 ".KQ" (KOSDAQ)
tickers = ["005930.KS", "000660.KS", ...]  # 시총 필터 통과 종목
data = yf.download(tickers, period="2y", group_by="ticker", threads=True)
# → 1회 호출로 전체 종목 OHLCV 획득
```

## 6. 모듈 구성

### 6.1 universe_builder.py — 유니버스 구축

- KRX Open API로 전 종목 시가총액 조회
- 시총 ≥ 1,000억원 필터
- 종목코드 + 종목명 + 시장(KOSPI/KOSDAQ) + 시총 반환
- yfinance 티커 변환: 코드 → `{코드}.KS` / `{코드}.KQ`

### 6.2 trend_analyzer.py — 5조건 판정 엔진

- 핵심 모듈: 5가지 조건 각각에 대한 Pass/Fail 판정
- 입력: 종목별 300일 OHLCV DataFrame
- 출력: 종목별 `{조건1: T/F, 조건2: T/F, ..., all_pass: T/F}`
- 함수:
  - `check_ma_trend(df, window=200, days=20)` → 조건 1
  - `check_ma_alignment(df)` → 조건 2
  - `check_52w_low_distance(df, threshold=0.30)` → 조건 3
  - `check_52w_high_distance(df, threshold=-0.25)` → 조건 4
  - `analyze_stock(df, config)` → 5조건 통합 판정

### 6.3 report_generator.py — 리포트 생성

- 통과 종목을 마크다운 테이블로 정리
- 섹션 구성:
  1. 요약 (통과 종목 수 / 전체 대상 종목 수)
  2. 통과 종목 테이블 (종목코드, 종목명, 시장, 시총, 현재가, 52주 고저 대비)
  3. 조건별 통과율 (각 조건 몇 %가 통과했는지)
  4. 실패 근접 종목 (4/5 통과 종목 = watch list)

### 6.4 kr_stock_selector.py — 오케스트레이터

- 전체 흐름 관리: 유니버스 → 분석 → 리포트
- fail-safe: 개별 종목 실패 시 skip
- 결과 dict 반환 + reports/ 저장
